# makefile testing homeassistant
## Home Assistant Custom Component Development Tips

### Makefile Best Practices
- Use single backslash (\) for line continuation, not double (\\)
- Test commands with 'make -n target' to see what would run without executing

### Testing Commands
- make venv - Create virtualenv
- make deps - Install dependencies  
- make test - Run full test suite with coverage
- make lint - Run linting checks
- make format - Auto-format code
- make typecheck - Run mypy type checking
- make clean - Remove build artifacts

### Home Assistant Integration Patterns
- async_add_executor_job() passes only positional args to the function
- For APIs requiring keyword args, wrap in lambda: lambda: api_call(kwarg=value)
- Config data stored in entry.data, behavioral options in entry.options
- Use manifest.json version as single source of truth for releases
